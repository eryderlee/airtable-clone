<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Create Neon project and provide connection strings</name>
  <action>
    The user must create a Neon project and provide connection strings. Claude cannot do this because it requires account creation and authentication in a web browser.
  </action>
  <instructions>
    1. Go to https://console.neon.tech and sign up or log in
    2. Click "New Project"
    3. Choose PostgreSQL version 15 or 16 (match your Supabase version)
    4. Select a region close to your Vercel deployment (us-east-1 recommended)
    5. Name the project "airtable-clone" (or any name you prefer)
    6. After creation, go to the project dashboard -> "Connection Details"
    7. Copy BOTH connection strings:
       - **Pooled** (has `-pooler` in hostname): This becomes DATABASE_URL
       - **Direct** (no `-pooler` in hostname): This becomes DIRECT_URL
    8. Both URLs should include `?sslmode=require` at the end (Neon adds this by default)

    Provide both connection strings when ready.
  </instructions>
  <resume-signal>Paste both Neon connection strings (pooled and direct)</resume-signal>
</task>

<task type="auto">
  <name>Task 2: Configure local env and apply schema to Neon</name>
  <files>src/server/db/index.ts, .env</files>
  <action>
    After receiving the Neon connection strings from Task 1:

    1. Update `.env` file:
       - Set `DATABASE_URL` to the Neon **pooled** connection string (with `-pooler` in hostname, `?sslmode=require`)
       - Set `DIRECT_URL` to the Neon **direct** connection string (without `-pooler`, `?sslmode=require`)
       - Keep all other env vars unchanged (AUTH_SECRET, AUTH_GOOGLE_ID, AUTH_GOOGLE_SECRET)

    2. Update `src/server/db/index.ts`:
       - Update the comment from "Supabase Supavisor (transaction pooler, port 6543)" to "Neon PgBouncer (transaction pooler)"
       - Keep `prepare: false` (still required for Neon PgBouncer transaction mode)
       - Keep `connect_timeout: 10`
       - Do NOT add explicit `ssl: 'require'` in the postgres() options — `?sslmode=require` in the connection URL is sufficient and Neon's default URLs include it. Only add `ssl: 'require'` as a fallback if SSL errors occur.

    3. Run `npx drizzle-kit push` to apply the schema to Neon
       - This uses DIRECT_URL (non-pooled) via drizzle.config.ts
       - All tables (users, accounts, sessions, verificationTokens, bases, tables, columns, rows, views) and indexes should be created

    4. Run `npm run dev` and verify the app starts without connection errors
       - Check the terminal for any SSL, connection timeout, or authentication errors
       - Navigate to http://localhost:3000 — it should load (may show sign-in page or empty state, both are fine)
  </action>
  <verify>
    - `npx drizzle-kit push` completes without errors and reports tables created/verified
    - `npm run dev` starts without connection errors in the terminal
    - Visiting http://localhost:3000 loads the app (no 500 errors)
  </verify>
  <done>
    Local dev environment connects to Neon successfully. Schema is applied. App loads without errors.
  </done>
