<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Update Vercel environment variables and redeploy</name>
  <action>
    The user must update Vercel env vars because they contain database credentials. Claude can assist with the Vercel CLI if the user is authenticated, but the credential values come from the Neon Console.
  </action>
  <instructions>
    **Option A: Via Vercel Dashboard (recommended)**

    1. Go to https://vercel.com -> airtable-clone project -> Settings -> Environment Variables
    2. Find `DATABASE_URL` -> Edit -> Replace value with your Neon **pooled** connection string (the one with `-pooler` in hostname, ending in `?sslmode=require`)
    3. Find `DIRECT_URL` -> Edit -> Replace value with your Neon **direct** connection string (without `-pooler`, ending in `?sslmode=require`)
    4. If `DIRECT_URL` does not exist, add it as a new variable (all environments: Production, Preview, Development)
    5. Leave all other env vars unchanged (AUTH_SECRET, AUTH_GOOGLE_ID, AUTH_GOOGLE_SECRET)
    6. Go to Deployments -> click "..." on the latest deployment -> "Redeploy" (or push any commit to trigger a new deploy)
    7. Wait for deployment to complete

    **Option B: Via Vercel CLI**

    If you have the Vercel CLI authenticated:
    ```bash
    # Remove old values
    vercel env rm DATABASE_URL production -y
    vercel env rm DIRECT_URL production -y

    # Add new Neon values (you will be prompted to paste the value)
    vercel env add DATABASE_URL production
    vercel env add DIRECT_URL production

    # Trigger redeploy
    vercel --prod
    ```

    Tell Claude when the deployment is complete and provide the deployment URL.
  </instructions>
  <resume-signal>Confirm "Deployed" and share any errors from the Vercel build log</resume-signal>
</task>
