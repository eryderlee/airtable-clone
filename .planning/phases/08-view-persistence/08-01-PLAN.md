<task type="auto">
  <name>Task 2: Auto-save config changes via debounced useEffect</name>
  <files>src/components/grid/GridView.tsx</files>
  <action>
1. Add a `view.updateConfig` tRPC mutation in GridView (no optimistic update needed -- fire and forget):

```tsx
const updateViewConfig = api.view.updateConfig.useMutation();
```

2. Add a debounced useEffect that saves filters, sorts, and hiddenColumns whenever they change. Use an `isFirstRender` pattern to avoid saving the initial values back (which would be a no-op but wastes a network call). Since we already have `isFirstRender` ref for the cache-reset effect, create a SEPARATE ref for this purpose (e.g., `isFirstConfigRender`):

```tsx
const isFirstConfigRender = useRef(true);

useEffect(() => {
  if (isFirstConfigRender.current) {
    isFirstConfigRender.current = false;
    return;
  }
  const timer = setTimeout(() => {
    updateViewConfig.mutate({
      id: viewId,
      config: { filters, sorts, hiddenColumns },
    });
  }, 800);
  return () => clearTimeout(timer);
}, [filters, sorts, hiddenColumns]); // eslint-disable-line react-hooks/exhaustive-deps
```

Key details:
- Do NOT include searchQuery or searchInput in the auto-save -- search is ephemeral (Phase 6 decision).
- The 800ms debounce prevents rapid-fire saves when user is actively toggling filters.
- The `isFirstConfigRender` ref prevents the initial mount from triggering a save (the useState already holds the correct values from SSR seeding).
- Do NOT add `viewId` or `updateViewConfig` to the dependency array -- viewId won't change within a mounted GridView instance (key={viewId} forces remount), and the mutation ref is stable.
