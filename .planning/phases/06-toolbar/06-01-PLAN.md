Modify the `getByOffset` procedure input schema to add three optional fields:
- `filters: z.array(filterConditionSchema).default([])`
- `sorts: z.array(sortConditionSchema).default([])`
- `searchQuery: z.string().default("")`

In the query handler, implement a TWO-PATH strategy:

**Fast path** (no filters, no sorts, no search): Keep the existing `rowOrder >= offset` seek. This is the current behavior and must remain unchanged for performance.

**General path** (any filter/sort/search active): Use true SQL OFFSET (not rowOrder seek, because sort order != rowOrder order when sorts are active).

```
if (input.sorts.length === 0 && input.filters.length === 0 && !input.searchQuery.trim()) {
  // Fast path: existing rowOrder seek (unchanged)
  // ...existing code...
} else {
  // General path:
  // 1. Build conditions array starting with tableId match
  // 2. Add filter conditions via buildFilterConditions(input.filters)
  // 3. Add search condition if searchQuery is non-empty: rows.cells::text ilike '%searchQuery%'
  // 4. Fetch column types for sort (reuse pattern from getRows)
  // 5. Build order clauses via buildSortOrder(input.sorts, columnTypeMap)
  //    If no sorts, default to asc(rows.rowOrder), asc(rows.id)
  // 6. Use .$dynamic() with .where(and(...conditions)).orderBy(...orderClauses).limit(input.limit).offset(input.offset)
  //    NOTE: .offset() here is SQL OFFSET, not rowOrder seek. This is O(n) but acceptable for Phase 6.
}
```

Also modify the `count` procedure input schema to add:
- `filters: z.array(filterConditionSchema).default([])`
- `searchQuery: z.string().default("")`

1. Build conditions starting with `eq(rows.tableId, input.tableId)`
2. Add `buildFilterConditions(input.filters)` results
3. Add search condition if searchQuery non-empty
4. Use `ctx.db.select({ count: count() }).from(rows).$dynamic().where(and(...conditions))`

CRITICAL: All new params have `.default([])` or `.default("")` so existing callers pass validation without changes.

Do NOT modify getRows — it already has filter/sort/search support. Only getByOffset and count need extension.

Update fetchPage to close over filters/sorts/searchQuery and pass them to getByOffset:
```typescript
const fetchPage = useCallback(
  async (pageIndex: number) => {
    if (pageCacheRef.current[pageIndex] !== undefined) return;
    if (loadingPagesRef.current.has(pageIndex)) return;
    loadingPagesRef.current.add(pageIndex);
    forceUpdate();
    try {
      const data = await utils.row.getByOffset.fetch({
        tableId,
        offset: pageIndex * PAGE_SIZE,
        limit: PAGE_SIZE,
        filters,
        sorts,
        searchQuery,
      });
      pageCacheRef.current[pageIndex] = data.items.map((r) => ({
        id: r.id,
        cells: r.cells,
      }));
    } finally {
      loadingPagesRef.current.delete(pageIndex);
      forceUpdate();
    }
  },
  [tableId, utils.row.getByOffset, filters, sorts, searchQuery],
);
```
