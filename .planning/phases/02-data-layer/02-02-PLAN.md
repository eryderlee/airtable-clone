**Zod schemas (define at top of file, export types for UI consumption):**

```typescript
// Filter operator schema — discriminated union on column type
const textFilterSchema = z.object({
  type: z.literal("text"),
  operator: z.enum(["contains", "does_not_contain", "equals", "is_empty", "is_not_empty"]),
  value: z.string().optional(), // not needed for is_empty/is_not_empty
});

const numberFilterSchema = z.object({
  type: z.literal("number"),
  operator: z.enum(["greater_than", "less_than"]),
  value: z.number(),
});

const filterConditionSchema = z.object({
  columnId: z.string().uuid(),
  filter: z.discriminatedUnion("type", [textFilterSchema, numberFilterSchema]),
});

const sortConditionSchema = z.object({
  columnId: z.string().uuid(),
  direction: z.enum(["asc", "desc"]),
});

// Export types
export type FilterCondition = z.infer<typeof filterConditionSchema>;
export type SortCondition = z.infer<typeof sortConditionSchema>;
```

**Helper: buildFilterConditions(filters: FilterCondition[]): SQL[]**

For each filter condition, generate a SQL fragment using `sql` template tag from drizzle-orm:
- `contains`: ``sql`${rows.cells}->>${f.columnId} ilike ${'%' + f.filter.value + '%'}` ``
- `does_not_contain`: ``sql`(${rows.cells}->>${f.columnId} is null or ${rows.cells}->>${f.columnId} not ilike ${'%' + f.filter.value + '%'})` ``
  Note: does_not_contain should also match rows where the cell is null (absence means it doesn't contain the value).
- `equals`: ``sql`${rows.cells}->>${f.columnId} = ${f.filter.value}` ``
- `is_empty`: ``sql`(${rows.cells}->>${f.columnId} is null or ${rows.cells}->>${f.columnId} = '')` ``
- `is_not_empty`: ``sql`(${rows.cells}->>${f.columnId} is not null and ${rows.cells}->>${f.columnId} != '')` ``
- `greater_than`: ``sql`CAST(${rows.cells}->>${f.columnId} AS numeric) > ${f.filter.value}` ``
- `less_than`: ``sql`CAST(${rows.cells}->>${f.columnId} AS numeric) < ${f.filter.value}` ``

IMPORTANT: The `->>` operator extracts JSONB value as text. The columnId is the UUID key in the JSONB object. Use `sql.raw(f.columnId)` for the key or pass it as a string literal in the template. Actually, Drizzle's `sql` template parameterizes values, but JSONB key access needs the key as a literal string. Use: ``sql`${rows.cells}->>'${sql.raw(f.columnId)}'` `` — BUT only after validating columnId is a valid UUID (the Zod schema already ensures this, so sql.raw is safe here).

**Helper: buildSortOrder(sorts: SortCondition[], columnTypeMap: Record<string, string>): SQL[]**

For each sort condition:
- If column type is "number": ``sql`CAST(${rows.cells}->>'${sql.raw(s.columnId)}' AS numeric) ${sql.raw(s.direction === "asc" ? "asc nulls last" : "desc nulls last")}` ``
- If column type is "text" (or unknown): ``sql`${rows.cells}->>'${sql.raw(s.columnId)}' ${sql.raw(s.direction === "asc" ? "asc nulls last" : "desc nulls last")}` ``

Always append `asc(rows.rowOrder), asc(rows.id)` as the final tie-breaker (this ensures stable cursor pagination).

**Procedure: row.getRows** — protectedProcedure, query:

Input schema:
```typescript
z.object({
  tableId: z.string().uuid(),
  cursor: z.object({
    rowOrder: z.number().int(),
    id: z.string().uuid(),
  }).optional(),
  limit: z.number().int().min(1).max(1000).default(100),
  filters: z.array(filterConditionSchema).default([]),
  sorts: z.array(sortConditionSchema).default([]),
  searchQuery: z.string().default(""),
  viewId: z.string().uuid().optional(),
})
```

Implementation steps:
1. **Verify ownership:** Inner join `tables` to `bases`, check `tables.id = input.tableId AND bases.userId = ctx.session.user.id`. Throw NOT_FOUND if not found.

2. **Resolve view config:** If `input.viewId` is provided, fetch the view. Merge: call-time params override stored view config. Specifically:
   - `effectiveFilters = input.filters.length > 0 ? input.filters : (view.config.filters as FilterCondition[])`
   - `effectiveSorts = input.sorts.length > 0 ? input.sorts : (view.config.sorts as SortCondition[])`
   - `effectiveSearch = input.searchQuery || view.config.searchQuery || ""`

3. **Fetch column types (only if sorts present):** Query columns for this table, build `Record<columnId, type>`.

4. **Build WHERE conditions array:**
   - Always: `eq(rows.tableId, input.tableId)`
   - If cursor: ``sql`(${rows.rowOrder}, ${rows.id}) > (${input.cursor.rowOrder}, ${input.cursor.id})` ``
     CRITICAL: Use ROW tuple comparison syntax. NOT the OR-expanded form `(rowOrder > X OR (rowOrder = X AND id > Y))`. The ROW form uses the composite index directly.
   - Filter conditions from `buildFilterConditions(effectiveFilters)`
   - Search: if effectiveSearch.trim() is non-empty: ``sql`${rows.cells}::text ilike ${'%' + effectiveSearch + '%'}` ``

5. **Build ORDER BY:** `buildSortOrder(effectiveSorts, columnTypeMap)` which includes the `(rowOrder, id)` tie-breaker.

6. **Execute query:**
   ```typescript
   const result = await ctx.db
     .select()
     .from(rows)
     .$dynamic()
     .where(and(...conditions))
     .orderBy(...orderClauses)
     .limit(input.limit + 1);
   ```
   CRITICAL: The `.$dynamic()` call after `.from(rows)` is REQUIRED. Without it, Drizzle's type system does not allow dynamically composed `.where()` and `.orderBy()` clauses built from runtime arrays. This is a hard architectural constraint — omitting `.$dynamic()` will cause type errors.

   Fetch `limit + 1` to determine if there's a next page.

7. **Compute nextCursor:**
   ```typescript
   const hasNextPage = result.length > input.limit;
   const items = hasNextPage ? result.slice(0, input.limit) : result;
   const nextCursor = hasNextPage && items.length > 0
     ? { rowOrder: items[items.length - 1]!.rowOrder, id: items[items.length - 1]!.id }
     : null;
   return { items, nextCursor };
   ```

**Procedure: row.create** — protectedProcedure, mutation:

Input: `{ tableId: z.string().uuid(), cells: z.record(z.string(), z.union([z.string(), z.number(), z.null()])).default({}) }`

1. Verify table ownership (same join pattern).
2. Get max rowOrder: `SELECT MAX(row_order) FROM rows WHERE table_id = input.tableId`. Use `sql<number>` for the aggregate. If null (empty table), use -1.
3. Insert row with `rowOrder: maxRowOrder + 1`.
4. Return created row.

**Procedure: row.update** — protectedProcedure, mutation:

Input: `{ id: z.string().uuid(), cells: z.record(z.string(), z.union([z.string(), z.number(), z.null()])) }`

1. Verify ownership: `rows INNER JOIN tables ON rows.tableId = tables.id INNER JOIN bases ON tables.baseId = bases.id WHERE rows.id = input.id AND bases.userId = ctx.session.user.id`. Fetch the current row.
2. Merge cells: `{ ...currentRow.cells, ...input.cells }` — this patches without overwriting unmodified keys.
3. Update the row with merged cells.
4. Return updated row.

**Procedure: row.delete** — protectedProcedure, mutation:

Input: `{ id: z.string().uuid() }`

1. Verify ownership (same 3-level join).
2. Delete row.
3. Return deleted row.
