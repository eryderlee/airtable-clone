  <action>
    **Create `src/components/nav/AppSidebar.tsx`:**
    - `"use client"` directive
    - Import `api` from `~/trpc/react` (classic createTRPCReact — NOT `useTRPC`)
    - Import `useRouter, usePathname` from `next/navigation`
    - Import `InlineEdit` from `~/components/ui/InlineEdit`
    - Import `useState` from `react`

    **Data fetching:**
    - `const { data: bases, isLoading } = api.base.getAll.useQuery()`
    - `const utils = api.useUtils()` (NOT `api.useContext()` — deprecated)

    **Mutations:**
    - `createBase`: `api.base.create.useMutation({ onSuccess: (newBase) => { void utils.base.getAll.invalidate(); router.push(`/base/${newBase.id}`); } })`
    - `renameBase`: `api.base.update.useMutation({ onSuccess: () => void utils.base.getAll.invalidate() })`
    - `deleteBase`: `api.base.delete.useMutation({ onSuccess: () => { void utils.base.getAll.invalidate(); router.push("/"); } })`

    **UI structure:**
    - `<aside className="flex w-64 flex-shrink-0 flex-col border-r border-gray-200 bg-white">` (matches Airtable sidebar width)
    - Header section: app name/logo + "+" button to create a new base
    - Create base: calls `createBase.mutate({ name: "Untitled Base" })`
    - Base list: map over `bases`, each item is a link to `/base/${base.id}`
    - Active state: use `usePathname()` — `pathname.startsWith(`/base/${base.id}`)` sets active highlight
    - Each base item has: `<InlineEdit value={base.name} onSave={(name) => renameBase.mutate({ id: base.id, name })} />` for inline rename
    - Delete: small "x" button or right-click context menu on each base item. Use a simple "x" button that appears on hover for now. Call `deleteBase.mutate({ id: base.id })`. Add a `window.confirm("Delete this base?")` guard before deletion.
    - Loading state: show skeleton or "Loading..." while `isLoading` is true

    **Update `src/app/(app)/layout.tsx`:**
    - Replace the placeholder `<aside>` with `<AppSidebar />`
    - Import: `import { AppSidebar } from "~/components/nav/AppSidebar"`
    - The layout remains a server component — AppSidebar is a client component imported into it (this is valid in Next.js App Router)
  </action>

  </verify>

    - `const params = useParams<{ tableId?: string }>()` — to determine active tab
    - `const utils = api.useUtils()`

    **Mutations:**
    - `createTable`: `api.table.create.useMutation({ onSuccess: async (newTable) => { ... } })`
      - After table create, resolve the auto-created viewId for navigation:
        1. Invalidate tables: `await utils.table.getByBaseId.invalidate({ baseId })`
        2. Fetch views for the new table: `const views = await utils.view.getByTableId.fetch({ tableId: newTable.id })`
        3. Navigate: `router.push(`/base/${baseId}/${newTable.id}/view/${views[0]!.id}`)`
      - Use `utils.view.getByTableId.fetch()` — this is the standard tRPC v11 imperative fetch on the `useUtils()` object. Do NOT use `utils.client` (that is a `@trpc/tanstack-react-query` concept, not available in classic `createTRPCReact`).
    - `renameTable`: `api.table.update.useMutation({ onSuccess: () => void utils.table.getByBaseId.invalidate({ baseId }) })`
    - `deleteTable`: `api.table.delete.useMutation({ onSuccess: () => { void utils.table.getByBaseId.invalidate({ baseId }); router.push(`/base/${baseId}`); } })`

    **Tab click navigation:**
    - Each tab links to `/base/${baseId}/${table.id}` via `<Link>`. The redirect page (below) resolves the viewId server-side.

    **Create `src/app/(app)/base/[baseId]/[tableId]/page.tsx`:**
    - Server component that redirects to the first view of the given table.
    - `params: Promise<{ baseId: string; tableId: string }>` — await it
    - Use server-side tRPC caller: `const views = await api.view.getByTableId({ tableId })`
    - Redirect: `redirect(`/base/${baseId}/${tableId}/view/${views[0]!.id}`)`
    - If no views (edge case), render "No views found" as a fallback

    - Map tables as tab items. Active tab: `params.tableId === table.id` gets `bg-blue-50 text-blue-700 font-medium` styling

    - "+" button at end to create new table: `createTable.mutate({ baseId, name: "Untitled Table", seed: true })`
    - Delete: small "x" on hover per tab, with `window.confirm` guard. Don't show delete if only 1 table.
