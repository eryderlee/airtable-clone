  <action>
    **Create `src/components/nav/TableTabBar.tsx`:**
    - `"use client"` directive
    - Props: `{ baseId: string }` — passed from the base layout
    - Import `api` from `~/trpc/react`, `useRouter, useParams` from `next/navigation`, `InlineEdit` from `~/components/ui/InlineEdit`

    **Data fetching:**
    - `const { data: tables } = api.table.getByBaseId.useQuery({ baseId })`
    - `const params = useParams<{ tableId?: string }>()` — to determine active tab
    - `const utils = api.useUtils()`

    **Mutations:**
    - `createTable`: `api.table.create.useMutation({ onSuccess: async (newTable) => { ... } })`
      - CRITICAL viewId resolution: After table create, the view was created server-side but the mutation only returns the table object. To get the viewId for navigation:
        1. Invalidate tables: `await utils.table.getByBaseId.invalidate({ baseId })`
        2. Fetch views for the new table: `const views = await utils.client.view.getByTableId.query({ tableId: newTable.id })`
        3. Navigate: `router.push(`/base/${baseId}/${newTable.id}/view/${views[0]!.id}`)`
      - If `utils.client` is not available in this tRPC setup, use `api.view.getByTableId.useQuery` with the newTable.id. Alternative: refetch using `utils.view.getByTableId.fetch({ tableId: newTable.id })` which returns a Promise.
      - Try `utils.view.getByTableId.fetch({ tableId: newTable.id })` first — this is the standard tRPC v11 way to do an imperative fetch.
    - `renameTable`: `api.table.update.useMutation({ onSuccess: () => void utils.table.getByBaseId.invalidate({ baseId }) })`
    - `deleteTable`: `api.table.delete.useMutation({ onSuccess: () => { void utils.table.getByBaseId.invalidate({ baseId }); router.push(`/base/${baseId}`); } })`

    **Tab click navigation:**
    - Each tab links to the table. But we need the viewId. Two approaches:
      - Option A: Use `<Link>` to `/base/${baseId}/${table.id}` which hits the `page.tsx` redirect that resolves viewId server-side. This is simpler and works because the page.tsx redirect is fast.
      - Option B: Fetch views on click and navigate directly.
    - USE OPTION A — simpler, and the server redirect is already implemented in Plan 01. The page.tsx at `base/[baseId]/page.tsx` handles resolving the first table+view. BUT wait — clicking a specific table tab should go to THAT table, not the first table. So the link href should be `/base/${baseId}/${table.id}` — but there's no page.tsx at that level to redirect. The `[tableId]` segment goes directly to `view/[viewId]`.
    - SOLUTION: Add a page at `src/app/(app)/base/[baseId]/[tableId]/page.tsx` that redirects to the first view of that table. OR, link directly to `/base/${baseId}/${table.id}/view/[firstViewId]` by pre-fetching views. Since tables query is already loaded, we can also pre-fetch views for all tables — but that's expensive.
    - BEST APPROACH: Create `src/app/(app)/base/[baseId]/[tableId]/page.tsx` as a server component that redirects to the first view. This keeps tab links simple as `/base/${baseId}/${table.id}`.

    **Create `src/app/(app)/base/[baseId]/[tableId]/page.tsx`:**
    - Server component, `params: Promise<{ baseId: string; tableId: string }>` — await it
    - Use server-side tRPC: `const views = await api.view.getByTableId({ tableId })`
    - Redirect to `redirect(`/base/${baseId}/${tableId}/view/${views[0]!.id}`)`
    - If no views (edge case), render "No views found"

    **UI structure:**
    - `<header className="flex h-10 flex-shrink-0 items-center gap-1 border-b border-gray-200 bg-white px-2">`
    - Map tables as tab items. Active tab: `params.tableId === table.id` gets `bg-blue-50 text-blue-700 font-medium` styling
    - Each tab: `<Link href={`/base/${baseId}/${table.id}`}>` wrapping `<InlineEdit>` for rename
    - "+" button at end to create new table: `createTable.mutate({ baseId, name: "Untitled Table", seed: true })`
    - Delete: small "x" on hover per tab, with `window.confirm` guard. Don't show delete if only 1 table.

    **Create `src/components/nav/ViewsPanel.tsx`:**
    - `"use client"` directive
    - Props: `{ tableId: string; activeViewId: string }` — passed from the view layout
    - `const { data: views } = api.view.getByTableId.useQuery({ tableId })`
    - Map views as a vertical list of links: `<Link href={`/base/${baseId}/${tableId}/view/${view.id}`}>`
    - BUT: ViewsPanel needs `baseId` for the link URL. Get it from `useParams<{ baseId: string }>()`.
    - Active view: `view.id === activeViewId` gets highlighted styling
    - For Phase 3, view creation is simple: a "+" button that creates a view named "Grid View N" via `api.view.create.useMutation()`. The view.create router accepts `{ tableId, name, config }`.
    - View creation config should be: `{ filters: [], sorts: [], hiddenColumns: [], searchQuery: "" }`

    **Update layouts:**
    - `src/app/(app)/base/[baseId]/layout.tsx`: Replace placeholder header with `<TableTabBar baseId={baseId} />`
    - `src/app/(app)/base/[baseId]/[tableId]/view/[viewId]/layout.tsx`: Replace placeholder aside with `<ViewsPanel tableId={tableId} activeViewId={viewId} />`
