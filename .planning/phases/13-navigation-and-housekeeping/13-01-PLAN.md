**TD-2: HomeContent.tsx cold-path fix (lines 119-132)**

The warm path (lines 107-118) must remain UNCHANGED. Only modify the cold path (the else branch starting at line 119).

Replace the current cold path:
```typescript
// Fallback: fetch fresh data client-side to avoid SSR race conditions
try {
  const fetchedTables = await utils.table.getByBaseId.fetch({ baseId });
  if (fetchedTables.length > 0 && fetchedTables[0]) {
    const fetchedViews = await utils.view.getByTableId.fetch({ tableId: fetchedTables[0].id });
    if (fetchedViews.length > 0 && fetchedViews[0]) {
      router.push(`/base/${baseId}/${fetchedTables[0].id}/view/${fetchedViews[0].id}`);
      return;
    }
  }
} catch {
  // If fetch fails, fall through to SSR redirect
}
router.push(`/base/${baseId}`);
```

With:
```typescript
// Cold path — navigate immediately, fetch in background
// The base layout will SSR-redirect to the correct table/view once data arrives
router.push(`/base/${baseId}`);
void (async () => {
  try {
    const fetchedTables = await utils.table.getByBaseId.fetch({ baseId });
    if (fetchedTables.length > 0 && fetchedTables[0]) {
      const fetchedViews = await utils.view.getByTableId.fetch({ tableId: fetchedTables[0].id });
      if (fetchedViews.length > 0 && fetchedViews[0]) {
        router.push(`/base/${baseId}/${fetchedTables[0].id}/view/${fetchedViews[0].id}`);
      }
    }
  } catch {
    // SSR redirect already handled navigation
  }
})();
```

Key points:
- `router.push(\`/base/${baseId}\`)` fires FIRST so URL changes immediately
- The async IIFE fetches data in the background and does a second `router.push` with the full path once data arrives
- The `handleBaseClick` function remains `async` but the `await` keyword is no longer blocking navigation
- Remove the `return` statements inside the cold path since the IIFE is fire-and-forget
