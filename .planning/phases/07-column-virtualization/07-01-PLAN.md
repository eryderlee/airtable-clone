**GridTable.tsx changes:**

1. Add a `COLUMN_VIRTUALIZATION_THRESHOLD = 20` constant at the top of the file.

2. Add a `columnVirtualizerRef` prop to `GridTableProps` (same pattern as `rowVirtualizerRef`):
   ```
   columnVirtualizerRef: React.MutableRefObject<Virtualizer<HTMLDivElement, Element> | null>;
   ```

3. Inside the `GridTable` component, after the existing `rowVirtualizer`, create a column virtualizer:
   ```typescript
   const shouldVirtualizeColumns = columnIds.length >= COLUMN_VIRTUALIZATION_THRESHOLD;

   const columnVirtualizer = useVirtualizer({
     count: columnIds.length,
     estimateSize: (index) => columnWidths[columnIds[index] ?? ''] ?? 180,
     getScrollElement: () => parentRef.current,
     horizontal: true,
     overscan: 3,
     enabled: shouldVirtualizeColumns,
   });
   columnVirtualizerRef.current = columnVirtualizer;
   ```

4. Compute derived values for rendering:
   ```typescript
   const virtualColumns = shouldVirtualizeColumns ? columnVirtualizer.getVirtualItems() : null;
   const virtualPaddingLeft = virtualColumns ? (virtualColumns[0]?.start ?? 0) : 0;
   const virtualPaddingRight = virtualColumns
     ? columnVirtualizer.getTotalSize() - (virtualColumns[virtualColumns.length - 1]?.end ?? 0)
     : 0;
   const columnsToRender: string[] = virtualColumns
     ? virtualColumns.map(vc => columnIds[vc.index]!)
     : columnIds;
   ```

5. Update the **real row** rendering (the `columnIds.map((colId) => ...)` loop inside the `rowVirtualizer.getVirtualItems().map()`) to:
   - Replace `columnIds.map(...)` with `columnsToRender.map(...)`.
   - Add a left padding spacer `<td>` before the column loop (only when `virtualPaddingLeft > 0`): `<td style={{ display: 'flex', width: virtualPaddingLeft }} />`.
   - Add a right padding spacer `<td>` after the column loop (only when `virtualPaddingRight > 0`): `<td style={{ display: 'flex', width: virtualPaddingRight }} />`.

6. Update the **skeleton row** rendering similarly — replace `columnIds.map(...)` with `columnsToRender.map(...)` and add the same left/right padding spacers.

7. Update the **"Add row" button row** at the bottom — replace `columnIds.map(...)` with `columnsToRender.map(...)` and add the same left/right padding spacers.

8. Update the **primary column border line** calculation. Currently it uses `columnIds[0]` to compute `left`. When column virtualization is active and the primary column is scrolled off-screen, this border line should still reference `columnIds[0]` (not `columnsToRender[0]`). The existing code is fine as-is since it reads from `columnIds[0]` — just verify it still works.

9. Pass `columnsToRender`, `virtualPaddingLeft`, and `virtualPaddingRight` to `GridHeader` as new props.

**GridHeader.tsx changes:**

1. Add new props to `GridHeaderProps`:
   ```typescript
   columnsToRender?: string[];  // column IDs to actually render (when virtualized)
   virtualPaddingLeft?: number;
   virtualPaddingRight?: number;
   ```

2. In the `GridHeader` component, when rendering the header `<tr>`:
   - After the checkbox `<th>`, add a left padding spacer `<th>` if `virtualPaddingLeft > 0`: `<th style={{ display: 'flex', width: virtualPaddingLeft, minWidth: virtualPaddingLeft, height: 32 }} className="border-b border-[#e2e0ea] bg-white" />`.
   - Replace the `headers.map(...)` loop: when `columnsToRender` is provided, filter `headers` to only include headers whose `id` (or `meta.columnId`) is in `columnsToRender`. When `columnsToRender` is not provided, render all headers as before (backward compatible).
   - After the headers loop and BEFORE the add-column `<th>`, add a right padding spacer `<th>` if `virtualPaddingRight > 0`: same pattern as left spacer but with `virtualPaddingRight` width.

**IMPORTANT pitfalls from research:**
- The row number column (first `<td>` with width 100) is NOT part of the column virtualizer. It is always rendered. `count: columnIds.length` only covers data columns.
- The add-column button `<th>` in the header goes AFTER the right padding spacer, not before it.
- `enabled: shouldVirtualizeColumns` on the virtualizer prevents any overhead for tables under 20 columns.

4. Update `scrollToCell` to include horizontal scrolling:
   ```typescript
   const scrollToCell = useCallback((rowIndex: number, columnId: string) => {
     const colIndex = visibleColumnIds.indexOf(columnId);
     requestAnimationFrame(() => {
       rowVirtualizerRef.current?.scrollToIndex(rowIndex, { align: "auto" });
       if (colIndex >= 0 && shouldVirtualizeColumns) {
         columnVirtualizerRef.current?.scrollToIndex(colIndex, { align: "auto" });
       }
       requestAnimationFrame(() => {
         const cellEl = document.querySelector<HTMLElement>(
           `[data-row-index="${rowIndex}"][data-column-id="${columnId}"]`,
         );
         cellEl?.focus();
       });
     });
   }, [visibleColumnIds, shouldVirtualizeColumns]);
   ```
