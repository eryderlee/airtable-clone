<task type="auto">
  <name>Task 2: Migrate HomeContent to useQuery and add optimistic base CRUD</name>
  <files>src/components/home/HomeContent.tsx</files>
  <action>
    1. **Fix the cache gap.** Replace the current pattern:
       ```tsx
       const bases = initialBases;
       ```
       With a React Query hook seeded by SSR data:
       ```tsx
       const utils = api.useUtils();
       const { data: bases } = api.base.getAll.useQuery(undefined, {
         initialData: initialBases,
       });
       ```
       This populates the React Query cache with the SSR data, enabling `utils.base.getAll.getData()` and `utils.base.getAll.setData()` to work.

    2. **Import toast** from `sonner`.

    3. **Create base — optimistic add.** Replace the current `createBase` pattern (if one exists, or wire the existing "Create base" button) with:
       ```tsx
       const createBase = api.base.create.useMutation({
         onMutate: async ({ name }) => {
           await utils.base.getAll.cancel();
           const previous = utils.base.getAll.getData();
           utils.base.getAll.setData(undefined, (old) => [
             ...(old ?? []),
             {
               id: `optimistic-${Date.now()}`,
               name,
               createdAt: new Date(),
               updatedAt: new Date(),
               lastOpenedAt: new Date(),
               color: null,
               userId: "",
             },
           ]);
           return { previous };
         },
         onError: (_err, _vars, context) => {
           if (context?.previous !== undefined) {
             utils.base.getAll.setData(undefined, context.previous);
           }
           toast.error("Failed to create base. Changes reverted.");
         },
         onSettled: () => {
           void utils.base.getAll.invalidate();
         },
       });
       ```
       Wire the "Create base" button in the EmptyState component and add a floating "+" action button (or header button) to call `createBase.mutate({ name: "Untitled base" })`. The user can then rename it via the rename flow.

    4. **Rename base — optimistic update-in-place.** Replace the current `renameBase` mutation:
       ```tsx
       const renameBase = api.base.update.useMutation({
         onMutate: async ({ id, name }) => {
           await utils.base.getAll.cancel();
           const previous = utils.base.getAll.getData();
           utils.base.getAll.setData(undefined, (old) =>
             old?.map((b) => (b.id === id ? { ...b, name: name ?? b.name } : b)) ?? [],
           );
           return { previous };
         },
         onError: (_err, _vars, context) => {
           if (context?.previous !== undefined) {
             utils.base.getAll.setData(undefined, context.previous);
           }
           toast.error("Failed to rename base. Changes reverted.");
         },
         onSettled: () => {
           void utils.base.getAll.invalidate();
         },
       });
       ```
       Remove the old `onSuccess: () => router.refresh()` — the optimistic update + invalidate replaces it.

    5. **Delete base — optimistic remove.** Replace the current `deleteBase` mutation:
       ```tsx
       const deleteBase = api.base.delete.useMutation({
         onMutate: async ({ id }) => {
           await utils.base.getAll.cancel();
           const previous = utils.base.getAll.getData();
           utils.base.getAll.setData(undefined, (old) =>
             old?.filter((b) => b.id !== id) ?? [],
           );
           return { previous };
         },
         onError: (_err, _vars, context) => {
           if (context?.previous !== undefined) {
             utils.base.getAll.setData(undefined, context.previous);
           }
           toast.error("Failed to delete base. Changes reverted.");
         },
         onSettled: () => {
           void utils.base.getAll.invalidate();
         },
       });
       ```
       Remove the old `onSuccess: () => router.refresh()`.

    6. **Remove `router.refresh()` calls.** The old mutations used `router.refresh()` to force a re-render from the server. This is no longer needed — `invalidate()` in `onSettled` re-fetches the cache, and the optimistic update handles the instant UI feedback.

    7. **Type alignment.** The `BaseRecord` type in HomeContent must include all fields that the optimistic object creates (`color`, `userId`). If the type doesn't already have those fields, add them as optional (or extend from the router's return type). Alternatively, use `as` assertion for the optimistic entry since it's temporary and gets replaced after invalidation.

    8. **Instant base navigation (client-side).** The base card's click/link handler currently navigates to `/base/[baseId]` which triggers an SSR redirect to the first table+view. Replace this with a client-side fast path:
       ```tsx
       const handleBaseClick = (baseId: string) => {
         const tables = utils.table.getByBaseId.getData({ baseId });
         if (tables && tables.length > 0) {
           const firstTable = tables[0]!;
           const views = utils.view.getByTableId.getData({ tableId: firstTable.id });
           if (views && views.length > 0) {
             router.push(`/base/${baseId}/${firstTable.id}/${views[0]!.id}`);
             return;
           }
         }
         // Fallback: SSR redirect (first visit or cold cache)
         router.push(`/base/${baseId}`);
       };
       ```
       Wire this handler to the base card's click event (replacing the existing `<Link href={...}>` or `onClick` navigation). This avoids the server round-trip when the user has already visited the base and table/view data is cached from a prior navigation. The `/base/[baseId]/page.tsx` SSR redirect remains as a fallback for direct URL access or cold cache.
