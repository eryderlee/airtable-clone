    1. Scaffold the project using create-t3-app:
       ```bash
       npm create t3-app@latest airtable-clone -- --CI --appRouter --trpc --drizzle --nextAuth --tailwind --dbProvider postgres
       ```
       Then move all scaffolded contents into the project root (since we are already in the repo root).

    2. Pin Zod to v3 immediately after scaffold:
       ```bash
       npm install zod@3
       ```
       Verify with `npm list zod` that version is 3.x (not 4.x).

    3. Install Auth.js v5 beta (not v4):
       ```bash
       npm install next-auth@beta @auth/drizzle-adapter
       ```
       Verify with `npm list next-auth` that version is 5.0.0-beta.x.

    4. Install supporting dev dependencies:
       ```bash
       npm install --save-dev @faker-js/faker tsx
       ```

    5. If the scaffold created `middleware.ts`, rename it to `proxy.ts`:
       ```bash
       npx @next/codemod@canary middleware-to-proxy .
       ```
       If the codemod fails or file is already named proxy.ts, skip.

    6. Update `src/server/db/index.ts` to use `prepare: false`:
       ```typescript
       import { drizzle } from "drizzle-orm/postgres-js";
       import postgres from "postgres";
       import * as schema from "./schema";

       const client = postgres(process.env.DATABASE_URL!, { prepare: false });
       export const db = drizzle({ client, schema });
       ```

    7. Update `drizzle.config.ts` to prefer DIRECT_URL for migrations:
       ```typescript
       import "dotenv/config";
       import { defineConfig } from "drizzle-kit";

       export default defineConfig({
         out: "./drizzle",
         schema: "./src/server/db/schema.ts",
         dialect: "postgresql",
         dbCredentials: {
           url: process.env.DIRECT_URL ?? process.env.DATABASE_URL!,
         },
       });
       ```

    8. Create `.env.example` with all required vars (no actual secrets):
       ```
       DATABASE_URL=postgresql://postgres.[ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres
       DIRECT_URL=postgresql://postgres.[ref]:[password]@aws-0-[region].supabase.com:5432/postgres
       AUTH_SECRET=
       AUTH_GOOGLE_ID=
       AUTH_GOOGLE_SECRET=
       ```

    9. Create `.env` with the same structure (user will fill in values).

    10. Update `src/env.js` (t3-env config) to include DIRECT_URL as an optional server env var. Keep existing T3 env validation intact.

    IMPORTANT: Do NOT install zod@4. Do NOT install next-auth@latest (that is v4). Always use next-auth@beta.
  </action>
  <verify>

    - `cat src/server/db/index.ts` contains `prepare: false`

    - `npx next build` completes without errors (may warn about missing env vars — that is OK at this stage)
  </verify>

  </done>
