  <action>
  Replace the chunk generation + parallel insert logic in `bulkCreate` with a single raw SQL query using Postgres `generate_series()`.

  The new approach:

  1. Keep the existing ownership check, maxOrder query, and columns fetch unchanged.

  2. Remove the faker import (no longer needed — data is generated in Postgres).

  3. Build a dynamic JSONB expression from the column list:
     ```typescript
     // Build per-column JSONB key-value pairs for json_build_object()
     // Column IDs are UUIDs from our own DB (safe to embed as literals)
     const jsonParts = cols.map((col) => {
       const colIdLiteral = `'${col.id}'`; // UUID — hex+hyphens only, no injection risk
       if (col.type === "number") {
         return `${colIdLiteral}, floor(random() * 10001)::int`;
       }
       // text: generate a space-separated "sentence" from 3 random words
       return `${colIdLiteral}, concat_ws(' ', substr(md5(random()::text),1,8), substr(md5(random()::text),1,8), substr(md5(random()::text),1,8))`;
     });

     const cellsExpr = jsonParts.length > 0
       ? `json_build_object(${jsonParts.join(", ")})::jsonb`
       : "'{}'::jsonb";
     ```

  4. Execute a single INSERT:
     ```typescript
     await ctx.db.execute(sql`
       INSERT INTO rows (id, table_id, row_order, cells)
       SELECT
         gen_random_uuid(),
         ${input.tableId}::uuid,
         ${startOrder} + (gs - 1),
         ${sql.raw(cellsExpr)}
       FROM generate_series(1, ${input.count}) AS gs
     `);
     ```

  5. Return `{ count: input.count }` as before.

  IMPORTANT constraints:
  - Column IDs are UUIDs fetched from our own `columns` table (hex + hyphens only). They are safe to embed as SQL string literals via `sql.raw()`. Do NOT parameterize them — they must be literal keys in `json_build_object()`.
  - The `sql` import from `drizzle-orm` is already in the file's imports.
  - `generate_series(1, N)` produces integers 1..N. Row order = startOrder + (gs - 1).
  - `gen_random_uuid()` is a Postgres built-in (Neon supports it).
  - `json_build_object()` with dynamic key/value pairs builds the JSONB cells object.
  - If cols is empty (no columns), use `'{}'::jsonb` as the cells value.
  - Remove the `@faker-js/faker` dynamic import from bulkCreate (it is no longer used here). Other procedures may still use faker — check before removing the import entirely. If only bulkCreate uses it, remove it from this procedure only (leave any other usages).
