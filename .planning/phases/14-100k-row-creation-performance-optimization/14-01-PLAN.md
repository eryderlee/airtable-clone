<objective>
Optimize 100k row creation from ~88 seconds to under 6 seconds by increasing chunk size from 1000 to 5000 rows and executing chunks in parallel batches of 5 concurrent inserts.

Purpose: The current sequential 1000-row chunks create 100 network round-trips (Vercel -> Neon PgBouncer -> Postgres -> back). Each round-trip averages ~880ms, totaling ~88s. Increasing chunk size to 5000 cuts round-trips to 20, and running 5 concurrently reduces to 4 sequential batches at ~880ms each = ~3.5s.

Output: Modified bulkCreate procedure and maxDuration export on the tRPC route handler.
</objective>

  2. In `src/server/api/routers/row.ts`, modify the `bulkCreate` procedure:

     a. Change CHUNK_SIZE from 1000 to 5000:
        ```typescript
        const CHUNK_SIZE = 5000;
        ```

     b. Add a CONCURRENCY constant:
        ```typescript
        const CONCURRENCY = 5;
        ```

     c. Replace the sequential for loop:
        ```typescript
        // OLD (sequential):
        for (let offset = 0; offset < input.count; offset += CHUNK_SIZE) {
          const chunkSize = Math.min(CHUNK_SIZE, input.count - offset);
          const chunk = Array.from({ length: chunkSize }, (_, i) => { ... });
          await ctx.db.insert(rows).values(chunk);
        }
        ```

        With a two-phase approach — pre-generate all chunks, then execute in parallel batches:
        ```typescript
        // Phase 1: Pre-generate all chunks
        const chunks: (typeof rows.$inferInsert)[][] = [];
        for (let offset = 0; offset < input.count; offset += CHUNK_SIZE) {
          const chunkSize = Math.min(CHUNK_SIZE, input.count - offset);
          const chunk = Array.from({ length: chunkSize }, (_, i) => {
            const cells: Record<string, string | number | null> = {};
            for (const col of cols) {
              if (col.type === "number") {
                cells[col.id] = faker.number.int({ min: 0, max: 10000 });
              } else {
                cells[col.id] = faker.lorem.words();
              }
            }
            return {
              tableId: input.tableId,
              rowOrder: startOrder + offset + i,
              cells,
            };
          });
          chunks.push(chunk);
        }

        // Phase 2: Execute in parallel batches of CONCURRENCY
        for (let i = 0; i < chunks.length; i += CONCURRENCY) {
          const batch = chunks.slice(i, i + CONCURRENCY);
          await Promise.all(batch.map((chunk) => ctx.db.insert(rows).values(chunk)));
        }
        ```

     d. Keep the rest of bulkCreate unchanged (ownership check, maxOrder query, columns query, return statement).

  IMPORTANT constraints:
  - Drizzle `.values()` with 5 columns at 5000 rows = ~25,000 params (under 32,767 limit). Do NOT exceed 5000 rows per chunk.
  - Neon free tier has ~10 pool connections. 5 concurrent is safe for single-user. Do NOT increase CONCURRENCY above 5.
  - The `typeof rows.$inferInsert` type hint ensures chunk arrays are type-safe. If this causes type errors, use `any[][]` instead (the shape is identical to the current code).
