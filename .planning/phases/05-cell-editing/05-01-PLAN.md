    **Create `src/components/grid/GridCell.tsx`:**
    - `"use client"` directive

    **Props interface:**
    ```typescript
    interface GridCellProps {
      rowId: string;
      rowIndex: number;
      columnId: string;
      columnType: "text" | "number";
      value: string | number | null;
      isFocused: boolean;
      isEditing: boolean;
      onCommit: (rowId: string, columnId: string, value: string | number | null) => void;
      onRevert: () => void;
      onStartEditing: () => void;
      onSelect: () => void;
    }
    ```

    **Local draft state:**
    - `const [draft, setDraft] = useState<string>("")` -- local edit value as string
    - When `isEditing` becomes true, initialize draft from current value: `useEffect` watching `isEditing` that sets `draft` to `String(value ?? "")`
    - The input ref: `const inputRef = useRef<HTMLInputElement>(null)` -- auto-focus the input when entering edit mode via a `useEffect` that calls `inputRef.current?.focus()` and `inputRef.current?.select()` when `isEditing` is true

    **Commit logic (helper function inside component):**
    ```typescript
    const handleCommit = useCallback(() => {
      if (columnType === "number") {
        const trimmed = draft.trim();
        const numValue = trimmed === "" ? null : Number(trimmed);
        // Only commit if valid number or null (empty)
        if (trimmed !== "" && isNaN(numValue as number)) return;
        onCommit(rowId, columnId, numValue);
      } else {
        onCommit(rowId, columnId, draft);
      }
    }, [draft, columnType, rowId, columnId, onCommit]);
    ```
    CRITICAL: Number columns save `null` for empty string, NOT `Number("")` which is `0`. Invalid number input (e.g., "abc" in a number column) is silently ignored (no commit).

    **Revert logic:**
    - Call `onRevert()` which will set `editingCell` to null in GridView (restoring navigation mode)
    - The draft is discarded because the component re-renders with `isEditing: false`

    **Click handler on the cell wrapper div:**
    - If not focused: call `onSelect()` (single click selects)
    - If focused but not editing: call `onStartEditing()` (second click enters edit mode)
    - This implements the Airtable two-click pattern: first click selects, second click edits

    **Keyboard handler on the INPUT element (only active in edit mode):**
    ```typescript
    const handleInputKeyDown = (e: React.KeyboardEvent<HTMLInputElement>) => {
      if (e.key === "Escape") {
        e.stopPropagation(); // Prevent container handler from double-handling
        onRevert();
      } else if (e.key === "Enter") {
        e.stopPropagation();
        handleCommit();
        // Note: 05-02 will add cursor movement after Enter commit
      } else if (e.key === "Tab") {
        // Tab is handled by the container-level handler in 05-02
        // For now, just commit on blur (the blur handler covers Tab)
      }
    };
    ```
    CRITICAL: `e.stopPropagation()` on Escape and Enter inside the input to prevent the container-level keyboard handler (added in 05-02) from also processing these keys.

    **Blur handler on input:**
    ```typescript
    const handleBlur = useCallback(() => {
      handleCommit();
    }, [handleCommit]);
    ```
    Blur commits the edit. This handles: clicking away, Tab (default browser behavior blurs before focusing next element).

    **Data attributes on the cell wrapper div:**
    ```typescript
    data-row-index={rowIndex}
    data-column-id={columnId}
    ```
    These are used for DOM-selector-based focus after scroll (the double-rAF pattern).

    **Render:**
    ```tsx
    <div
      data-row-index={rowIndex}
      data-column-id={columnId}
      tabIndex={-1}
      onClick={handleClick}
      className={`flex h-full w-full items-center truncate px-2 py-1 text-sm outline-none ${
        isFocused
          ? "ring-2 ring-inset ring-blue-500"
          : ""
      }`}
    >
      {isEditing ? (
        <input
          ref={inputRef}
          type={columnType === "number" ? "number" : "text"}
          value={draft}
          onChange={(e) => setDraft(e.target.value)}
          onKeyDown={handleInputKeyDown}
          onBlur={handleBlur}
          className="h-full w-full bg-transparent outline-none"
        />
      ) : (
        <span className="truncate">{value ?? ""}</span>
      )}
    </div>
    ```
    - `tabIndex={-1}` allows programmatic focus but not Tab focus (Tab is managed by keyboard handler)
    - `ring-2 ring-inset ring-blue-500` for the blue selection ring (Airtable style)
    - Number inputs use `type="number"` for native browser validation UX

    **Export:** Named export `GridCell`.

    These live at the GridView level because the keyboard handler (05-02) and optimistic mutation (05-02) both need access.

    **Add a ref for the rowVirtualizer (needed for scrollToIndex from GridView):**
    ```typescript
    const rowVirtualizerRef = useRef<Virtualizer<HTMLDivElement, Element> | null>(null);
    ```
    Import `type Virtualizer` from `@tanstack/react-virtual`.

    **scrollToCell helper function:**
    ```typescript
    const scrollToCell = useCallback((rowIndex: number, columnId: string) => {
      requestAnimationFrame(() => {
        rowVirtualizerRef.current?.scrollToIndex(rowIndex, { align: "auto" });
        requestAnimationFrame(() => {
          const cellEl = document.querySelector<HTMLElement>(
            `[data-row-index="${rowIndex}"][data-column-id="${columnId}"]`
          );
          cellEl?.focus();
        });
      });
    }, []);
    ```
    CRITICAL: Double requestAnimationFrame is REQUIRED. The first rAF allows the virtualizer to scroll and render. The second rAF allows the newly rendered cell to be queryable in the DOM.

    **onSelect handler:**
    ```typescript
    const handleSelect = useCallback((rowIndex: number, columnId: string) => {
      setCursor({ rowIndex, columnId });
      setEditingCell(null); // Exit edit mode on new selection
      scrollToCell(rowIndex, columnId);
    }, [scrollToCell]);
    ```

    **onStartEditing handler:**
    ```typescript
    const handleStartEditing = useCallback((rowIndex: number, columnId: string) => {
      setEditingCell({ rowIndex, columnId });
    }, []);
    ```

    **onRevert handler:**
    ```typescript
    const handleRevert = useCallback(() => {
      setEditingCell(null);
    }, []);
    ```

    const handleCommit = useCallback((rowId: string, columnId: string, value: string | number | null) => {
